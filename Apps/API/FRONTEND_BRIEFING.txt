Briefing único para implementar o Frontend (novas entidades)

Base e gerais
- Base da API: http://localhost:5000
- Swagger: http://localhost:5000/swagger
- Formato: JSON
- CORS permitido: http://localhost:5173, http://localhost:3000, http://localhost:4173

Modelos e enums (TypeScript)
// Equipes
export type AreaEquipe = 1 | 2 | 3; // 1=Verde, 2=Vermelha, 3=Laranja

export type Equipe = {
  id: number;
  nome: string;
  area: AreaEquipe;
  dataCriacao: string; // ISO
};

export type CriarEquipe = { nome: string; area: AreaEquipe };
export type AtualizarEquipe = CriarEquipe;

// Cargos
export type Cargo = {
  id: number;
  nome: string;
  dataCriacao: string; // ISO
};

export type CriarCargo = { nome: string };
export type AtualizarCargo = CriarCargo;

// Voluntários
export type Voluntario = {
  id: number;
  nome: string;

  equipeId: number;
  nomeEquipe: string;   // vindo da API
  cargoId: number;
  nomeCargo: string;    // vindo da API
  dataCadastro: string; // ISO
};

export type CriarVoluntario = {
  nome: string;
  whatsApp: string;
  email?: string | null;
  equipeId: number;
  cargoId: number;
};

export type AtualizarVoluntario = CriarVoluntario;

Endpoints (REST)
- Equipes: /api/equipes
  - GET → Equipe[]
  - GET /{id} → Equipe
  - POST (body: CriarEquipe) → Equipe
  - PUT /{id} (body: AtualizarEquipe) → Equipe
  - DELETE /{id} → 204
- Cargos: /api/cargos
  - GET / GET /{id} / POST / PUT /{id} / DELETE /{id}
- Voluntários: /api/voluntarios
  - GET → Voluntario[]
  - GET /{id} → Voluntario
  - POST (body: CriarVoluntario) → Voluntario
  - PUT /{id} (body: AtualizarVoluntario) → Voluntario
  - DELETE /{id} → 204

Páginas/Fluxos a implementar
- Equipes
  - Lista com busca por nome e filtro por Área (dropdown 1/2/3 com labels “Verde/Vermelha/Laranja”).
  - Criar/Editar (modal ou página): campos nome (obrigatório), area (obrigatório).
  - Remover com confirmação (exibir erro caso haja vínculos).
- Cargos
  - Lista; Criar/Editar (validação nome obrigatório); Remover com confirmação.
- Voluntários
  - Lista com busca por nome e filtros por Equipe e Cargo (opções carregadas de /api/equipes e /api/cargos).
  - Criar/Editar:
    - nome (obrigatório)
    - whatsApp (obrigatório, máscara/validação numérica)
    - email (opcional, validar formato)
    - equipeId (Select obrigatório)
    - cargoId (Select obrigatório)
  - Tabela deve exibir nome, whatsApp, nomeEquipe, nomeCargo.
  - Remover com confirmação.

Validações sugeridas
- nome: obrigatório; ideal ≤100 chars.
- whatsApp: apenas dígitos; 10–13 dígitos (regex: ^\d{10,13}$).
- email: formato válido quando preenchido.
- area: aceitar somente 1 | 2 | 3.

UX/Estados
- Carregamento: skeleton/spinner ao buscar Equipes/Cargos para os selects.
- Erros:
  - Exibir mensagens do backend (400/404/409) via toast/alert.
  - Em DELETE de Equipe/Cargo com vínculos, mostrar “Existe(m) voluntário(s) vinculado(s)”.
- Empty states: mensagens amigáveis quando listas vazias.
- Paginação: opcional; client-side por enquanto.

Integração (client API)
- Headers: Content-Type: application/json
- Clients recomendados (um por recurso) com métodos: list(), get(id), create(dto), update(id, dto), remove(id).
- Cache:
  - React: @tanstack/react-query (invalidar queries após mutações).
  - Blazor: revalidar listas pós-cadastro/edição/remoção.

Critérios de aceite (mínimos)
- Criar Equipe e Cargo.
- Criar Voluntário selecionando Equipe e Cargo existentes.
- Listas atualizadas após criar/editar/remover.
- Filtros de Voluntários por Equipe e Cargo funcionando.
- Validações e toasts/erros exibidos adequadamente.
